Webpack 基础概念

发表于 2023-06-21|更新于2023-07-18|工程化
|字数总计:1976|阅读时长:7分钟|阅读量:

Webpack

  • 现代 JavaScript 应用程序的静态模块打包器(module bundler)
  • 能够识别遵循ES2015的模块语法、CommonJS、amd编写的模块
  • 递归地构建一个依赖关系图(dependency graph)
  • 构建 bundle

五个核心概念

  • 入口 entry
    • 指示 webpack 应该使用哪个模块，来作为构建其内部依赖图的开始
  • 输出 output
    • 告诉 webpack 在哪里输出它所创建的 bundles，以及如何命名这些文件
  • loader
    • 将非JavaScript 文件 转换 为 webpack 能够处理的有效模块 在 import 或"加载"模块时预处理文件
  • 插件 plugin
    • 打包优化、压缩、定义环境中的变量等各种各样的任务
    • 解决 loader 无法实现的其他事
  • 模式 mode
    • 启用 development/production 模式下的配置

基础配置

  • 常用命令

    • webpack --stats detailed
    • webpack -c webpack.config.dev.js

  • 资源模块 Asset Module

    • 类型 Type
      • asset/resource： 发送一个单独的文件并导出 URL
      • asset/inline：导出资源的 Data URL（Base64）
      • asset/source：导出资源源代码
      • asset：导出URL和发送一个单独的文件之间自动进行选择（通用资源类型）

      • 方式一
      • 方式二

    • 动态导入：通过模块内联函数调用来分离代码

      • import(/* webpackChunkName: ‘name’ */ ‘name.js’)

    • 预获取/预加载模块[懒加载]

      • prefetch：将来某些导航下需要的资源
        • import(/* webpackChunkName: ‘name’, webpackPrefetch: true */ ‘name.js’)
        • 告诉 webpack 执行预获取，生成 并加载到页面头部，提示浏览器在空闲时间 获取 name.js
      • preload：当前导航下需要的资源
        • import(/* webpackChunkName: ‘name’, webpackPreload: true */ ‘name.js’)

课堂疑问

  1. chunkhash / contenthash 区别
  2. yarn 与 npm 的区别
  3. 为何host 设置 ‘0.0.0.0’ 其他电脑即可访问你的页面？
  4. dev 模式下 index.html 如何引用内存中的bundles.js
  5. 热替换与热加载区别

问题一：

  1. hash ：表示所有文件哈希值相同，如果任意文件内容发生变化，则再次打包后，所有哈希值均改变且相同。即当任意module发生改变时，所有bundle的hash都改变且相同。
  2. chunkhash ：根据不同的入口文件(Entry)进行依赖文件解析，构建对应的chunk，生成对应的哈希值。
  3. contenthash :在打包时，我们会在js文件中导入CSS文件，因为他们是同一个入口文件，我们只改了JS代码，但是它的CSS在抽离生成CSS文件时hash也会跟着变，这个时候就需要contenthash来解决。

问题二：
yarn

  1. 有版本锁定文件yarn-lock.json
  2. 离线模式下从本地缓存安装依赖(并发下载)
  3. 输出日志简洁，语义化下载命令

npm

  1. 以队列模式下载依赖，下载错误后仍继续下载
  2. 日志输出复杂繁多
  3. 5.0版本后采用symlinks 链接本地缓存(不进行文件拷贝)，加快安装速度

问题四：

  • localhost也叫local 本地服务器
  • 127.0.0.1 ～本机地址(本机服务器)
    他们的解析通过本机的host文件，windows自动将localhost解析为127.0.0.1
    • 但是本机IP与127.0.0.1的区别是：
      • 127.0.0.1只能通过本机访问
      • 本机IP通过本机访问也能通过外部访问
  • 0.0.0.0代表本机可访问的所有IP地址

问题五：
热刷新(hot)：文件改动后，整个页面刷新，不保留任何状态（比如输入过内容的Input表单），相当于webpack帮你摁了F5刷新
热加载(liveReload)：文件改动后，以最小的代价改变被改变的区域。尽可能保留改动文件前的状态（对input输入内容后，修改其他标签的代码）
